# PHASE 2 — GUARDIAN PORTAL

## Objective

Build a safe guardian read portal using Phase 1 ownership and posting boundaries, without introducing live payment gateway logic yet.

## Scope

- guardian route group
- guardian dashboard
- linked student profile visibility
- invoice list
- receipt/payment history visibility
- ownership policies and tests

## Hard Restrictions

- guardian must only access linked students and linked records
- no payment gateway integration yet
- no route-name rewrites in existing management UI
- no broad report changes
