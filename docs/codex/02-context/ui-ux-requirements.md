# UI/UX Requirements

- Use one global-standard, modern, usability-first design direction.
- Do not treat the existing inconsistent UI as the target pattern.
- Donor, guardian, public, auth, and multi-role surfaces must feel like one product family.
- Standardize layout, spacing, typography, button hierarchy, forms, cards, lists, tables, empty states, loading states, validation states, and feedback states.
- Accessibility, responsiveness, readability, and mobile usability are first-class requirements.
- Include a UI standardization phase before fragmented feature-page work.
