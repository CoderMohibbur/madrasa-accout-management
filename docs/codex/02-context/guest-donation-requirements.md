# Guest Donation Requirements

## Target rule
Donation must not require prior registration.

## Supported behaviors
- Guest donation allowed.
- Direct amount-based donation allowed.
- Name, phone, and email may be optional.
- Optional contact data may later support lightweight account creation or identity attachment.
- If no usable identity data is provided, donation must still be recorded safely using approved terminology.

## Terminology to standardize
- Guest donor
- Anonymous-display donor
- Hidden donor
- Identified donor

## Safety requirements
- Guest donation must not require portal access.
- Payment traceability must be preserved.
- Anti-abuse controls must be preserved.
- Reconciliation and reporting safety must be preserved.
- Unsafe legacy transaction shortcuts must not be used for guest donation finalization.
