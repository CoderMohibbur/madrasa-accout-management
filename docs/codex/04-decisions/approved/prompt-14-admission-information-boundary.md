# Prompt 14 Admission Information Boundary

Approved baseline decisions from prompt-14:

- Admission scope is informational-only with an external application handoff; this Laravel app must not become a full admission application system.
- Public admission information may include only generic institution/admission content plus the external application CTA.
- Authenticated admission information belongs only inside the light guardian informational portal and may add self-only onboarding/help/status messaging, not protected or workflow-owned admission data.
- The external admission button/link is an explicit handoff boundary and must remain separate from guardian protected access, donor flows, and internal student/invoice/payment data.
- One canonical external admission destination should later be reused across approved placements instead of duplicating hard-coded links.
- Prompt-13 guardian informational/protected separation remains preserved and is not reopened by prompt-14.
