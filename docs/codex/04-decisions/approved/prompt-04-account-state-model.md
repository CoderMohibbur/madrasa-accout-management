# Prompt 04 Account State Model

Approved baseline decisions from prompt-04:

- Replace the single overloaded activation concept with orthogonal account-state axes.
- Keep approval, email verification, phone verification, role assignment, profile existence, portal eligibility, linkage, lifecycle, and deletion as separate state dimensions.
- Treat guardian informational access and guardian protected access as distinct derived states.
- Keep guest donor identity separate from registered donor identity.
- Do not let deletion or suspension remain implicit profile-only behavior.
