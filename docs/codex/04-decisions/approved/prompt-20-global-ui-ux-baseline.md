# Prompt 20 Global UI/UX Baseline

Approved baseline decisions from prompt-20:

- The product-wide UI direction is a light-first institutional design system: calm, trustworthy, readable, and operationally clear across public, auth, portal, and management surfaces.
- Donor, guardian, management, auth, public, and future multi-role views must feel like one product family; role differences should be expressed through small accent tokens and content, not separate design languages.
- The donor and guardian portal information architecture is worth preserving, but its shell and components must be normalized into shared primitives rather than maintained as duplicated layouts.
- The current public welcome page, stock Breeze auth styling, and page-local token definitions are not the target standard and should be replaced by the shared baseline in later implementation prompts.
- The minimum shared design foundation must include semantic tokens, shared shells, buttons, forms, alerts, cards, tables, empty states, loading states, and no-access states before feature-page UI work expands.
- Prompt-19 remains preserved: any future admission CTA styling must keep the handoff external-only and limited to approved public and guardian-informational placements.
