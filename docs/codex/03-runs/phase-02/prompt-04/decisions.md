# Decisions

- Freeze the target account model as orthogonal state axes rather than a single overloaded activation field.
- Treat admin approval, email verification, phone verification, account activity, portal eligibility, and guardian linkage as separate concerns that later prompts must not collapse back together.
- Freeze guardian informational access and guardian protected access as two different eligibility states.
- Freeze guest donor identity as a distinct state that does not require or imply a registered user account.
- Treat current legacy management compatibility as an interim derived eligibility rule, not a permanent role model.
